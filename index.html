ДЗ-1

Атрибуты async и defer применяются для распараллеливания загрузки ресурсов сайта. Конкретно для запуска механизма загрузки скриптов параллельно загрузке DOM. В классическом варианте DOM не отобразится пока не будут загружены все скрипты, при этом DOM будет загружаться последовательно и дойдя до строчки с указанием внешнего скрипта перейдёт к его загрузке и после его загрузки продолжин загружать DOM, при этом скрипту не будут доступны элементы DOM расположенные ниже него (пока эти элементы не прогрузятся). В настоящий момент скрипты могут быть очень большими и это вызывает большие задержки отображения DOM у пользователя. Атрибуты async и defer вносят дополнительные карты в колоду WEB разработчиков. Если указать в теге script аттрибут async браузер будет загружать данный скрипт параллельно DOM, но исполнит скрипт только после полной загрузки DOM + до события DOMContentLoaded. При этом скрипты обозначенные defer будут исполняться в той последовательности в которой они описаны в DOM — скрипты могут загрузиться раньше DOM и в разном порядке, но будут исполнены после полной загрузки DOM и в той последовательности в которой они определены в DOM. Аттрибут async применяется если скрипт является полностью автономным и независимым от DOM — загрузится и исполнится в неизвестной последовательности, очерёдность загрузки и исполнения для скриптов помеченных аттрибутом async не соблюдается.

A: a = 1, b = 1, c, d;
B: c = ++a; alert(c); // 2
C: d = b++; alert(d); // 1
D: c = (2+ ++a); alert(c); // 5
E: d = (2+ b++); alert(d); // 4
F: alert(a); // 3
G: alert(b); // 3
Почему код даёт именно такие результаты?

В строке B выводится 2, т.к. переменной "с" присваивается "а" с прединкрементом, т.е. сначала будет произведено увеличение "а" на 1 и результат инкремента присвоится переменной "с". Так как переменной "a" присвоено значение 1 при инициализации, то alert отобразит 2.
После этой строки скрипта "a"=2, "c"=2.

В строке C выводится 1, т.к. переменной "d" присваивается "b" с постинкрементом, т.е. сначала будет произведено присваивание "d" значения перменной "b", а после этого "b" увеличится на 1. Так как переменной "b" присвоено значение 1 при инициализации, то alert отобразит 1.
После этой строки скрипта "b"=2, "d"=1.

В строке D выводится 5, т.к. к двум прибавляется переменная "а" с прединкрементом, сначала "а" будет увеличено на 1 и спосле этого будет произведено сложение результата с 2. Переменная "а" после строки B скрипта стала равна 2. Таким образом производится сложение 2 и 3, что равно 5 и это значение присваивается переменной "с", а переменная "а" увеличивается на 1.
После этой строки скрипта "a"=3, "c"=5.

В строке E выводится 4, т.к. к двум прибавляется переменная "b" с постинкрементом (увеличение "b" будет произведено после операции сложения с 2). Переменная "b" после строки C скрипта стала равна 2. Таким образом производится сложение 2 и 2, что равно 4 и это значение присваивается переменной "d", а переменная "b" увеличивается на 1.
После этой строки скрипта "b"=3, "d"=4.

В строках F и G выводится 3, исходя из предыдущих объяснений - это значения переменных "a" и "b" в этой точке скрипта.
